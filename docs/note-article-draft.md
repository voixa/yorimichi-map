## 街歩きガチャとは

吉祥寺・谷根千・神保町・浅草エリアのキュレーション済み散歩コースを、**ガチャ形式で1本だけ引いて**スタンプラリーで遊べるWeb地図サービスです。

### 体験フロー
1. 巨大なガチャマシンのハンドルを回す
2. 当選コース（たとえば「霧の井の頭・早朝禁忌の弁天さま」）を地図に表示
3. GPSで自動チェックインしながら散歩
4. 全スポットを巡ると完走証明書が発行
5. SNSで自慢

無料、登録不要、PWAでスマホのホーム画面にも追加できます。

---

## なぜ作ったか

3つの動機がありました。

### 1. 既存の地図アプリへの不満
Googleマップは便利すぎて「**偶然の出会い**」がない。検索したものしか出てこない。散歩は本来「予定外の発見」が楽しいはず。

### 2. ガチャ的体験の汎用性
位置情報×ゲーム性は世界の歩き方を変えた。同じ仕組みを「**大人向けの散歩**」に応用したらどうなるか試したかった。

### 3. Claude Code で「個人で何ができるか」の実験
最近話題のAIコーディング。VOIXAの本業もあるので、副業として **どこまで爆速で立ち上げられるか** 検証したかった。

## 技術スタック

完全に静的なWebアプリです。

- **フロント**：Vanilla HTML/CSS/JavaScript（フレームワークなし）
- **地図**：Leaflet.js + OpenStreetMap タイル
- **経路計算**：OSRM 公開API
- **ジオコーディング**：Nominatim
- **ホスティング**：Google Cloud Run + Docker（nginx alpine）
- **PWA**：Service Worker + manifest.json
- **音声**：Web Speech API
- **GPS**：Geolocation API（watchPosition）

バックエンドゼロ。すべてブラウザで完結。データは localStorage に保存。

## Cloud Runでの公開

VOIXAも Cloud Run で動いてるので、**コマンド1つで公開**しました。

```bash
gcloud run deploy yorimichi-map --source . --region asia-northeast1 --allow-unauthenticated
```

Cloud Build がDockerイメージを作ってデプロイ。HTTPS・グローバル配信・スケーリング全部自動。月額コストは無料枠内で**ほぼ¥0**。

## ハマりどころ

### 1. 近すぎるスポットでルートがジグザグ
OSRMが実際の道に沿ったルートを返すので、座標が近いスポット間で細い道を行ったり来たり。座標を意図的にずらして解決。

### 2. Service Worker のキャッシュ
PWAの宿命。デプロイしても古い版が残る。SW のキャッシュキー（`yorimichi-v1` → `v8`）を更新するルールを徹底。

### 3. マーカーが番号だけだと分かりにくい
最初は番号「①②③」だけだったが、ユーザーから「どれが何のスポットか分からない」という声。常時表示の名前ラベルを追加。

## ビジネス展開

現状は無料公開で**バリデーション中**。今後検討している収益モデル：

### B2C
- ガチャ引き直しのコイン課金（¥100〜¥5,000）
- 1日3回の無料引き直し → それ以上は課金

### B2B
- **商店街振興組合との提携**：公式コース掲載・送客手数料
- **自治体・観光協会**：地方創生交付金活用での公式採用
- **個店の割引クーポン**：来店促進＋手数料モデル

将来的には「**コース内の店舗で割引**」機能を実装し、街歩きガチャ加盟店ネットワークを作るのが目標です。

## 「動画では言ってないこと」と「やってみて分かったこと」

最近 **「Claude Code起業入門」** のような動画が増えていますが、実際にやってみて感じた現実：

### 良かった点
- ✅ 個人で月の小遣い程度のサービスは実現可能
- ✅ ピボットコストが激安。「気に入らなかったら作り直す」が現実的になった
- ✅ アイデア検証スピードが100倍違う

### しんどい点
- ⚠️ 集客は別問題。コードができてもバズらない
- ⚠️ コンテンツ作業（コース内容のキュレーション）はAIで代替できない
- ⚠️ Service Worker・PWAなど、初心者がハマりやすいポイントは結局自分で読み解く必要がある
